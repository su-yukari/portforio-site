ローリングストック×育成ゲーム

UX/UI

Figma
planning | design | user test

20代女性が、日常の中で楽しく防災を続けられるUX設計

Overview

-Project

  • チーム制作（3人）
  • 期間：3ヶ月
  • 担当領域：課題設定 / コンセプト設計 / ペルソナ定義 / ユーザー体験設計 / UIデザイン

-Goal

ローリングストックリストを活用しながら、【防災の知識と備えを楽しく身につけられる体験】を作る

-Process

1. なぜこの課題に取り組んだのか

防災は「大切だと分かっているのに続かない」行動の代表例です。 多くの防災アプリや情報は、知識提供やチェックリストが中心で、ユーザーの生活リズムや感情に寄り添えていないと感じました。 災害はいつ起こるか分からない一方で、日常では実感しづらく、結果として備えが後回しになります。 この“意識と行動のズレ”をUXの力で埋められないかと考えたことが、本プロジェクトの出発点です。

2. 試行錯誤/課題の再定義

テーマ設定の難しさの直面

防災というテーマは非常に範囲が広く、当初は「何を作るべきか」を絞り込むことに大きな難しさがありました。
ペルソナ設定においても、年齢やライフスタイル、災害経験の有無など条件が多く、方向性が定まらない時期が続きました。
その中で、「役に立つ機能」を増やすだけでは行動につながらないことに気づき、ユーザーの日常行動そのものに目を向け直しました。

3. 気づき

防災は「特別な行動」ではなく「日常の延長」にする必要がある

防災というテーマは非常に範囲が広く、当初は「何を作るべきか」を絞り込むことに大きな難しさがありました。
ペルソナ設定においても、年齢やライフスタイル、災害経験の有無など条件が多く、方向性が定まらない時期が続きました。
その中で、「役に立つ機能」を増やすだけでは行動につながらないことに気づき、ユーザーの日常行動そのものに目を向け直しました。
その試行錯誤の末、”管理ではなく「育てる」「触れる」体験に置き換える” ことからローリングストックという考え方に行きつきました。

4. コンセプト定義・ペルソナ設定

そだてて そなえて いっしょに安心。

防災への関心はあるものの、具体的な知識や経験は少なく、「何から始めればいいか分からない」状態にあります。
ペルソナは〔20代OL女性 / 被災経験はない〕という層に焦点を当て、仕事や私生活の忙しさ・日用品の買い足し頻度などを深掘り、漠然とした不安があるという心理状態を前提に設計しました。

カスタマージャーニーマップでペルソナの行動を分析

5. UX設計｜ユーザー体験の流れ

  1. キャラクターと出会う
  2. ストックリストを登録・確認
  3. ストック管理と育成でキャラクターが成長
  4. 備えの習慣ができる

ユーザーはまずキャラクターと出会い、簡単なストック登録から体験が始まります。
日用品や食品を補充・消費するたびにキャラクターが反応し、成長していくことで行動の結果が可視化されます。
成長段階に応じて、防災の範囲も「家の中」から「外出時」へと広がり、無理なく知識と行動が積み重なる設計としました。

6. デザイン

■ ■ ■

  • 防災の堅苦しさ・緊張感を感じさせない色・トーン
  • 余白を活かした安心感のあるレイアウト
  • キャラクターとの距離が近い設計

UIデザインでは、防災特有の堅苦しさや緊張感を感じさせないことを重視しました。
柔らかい色味、シンプルなレイアウト、余白を活かした構成により、安心感と親しみやすさを表現しています。
キャラクターとの距離を近く感じられるよう、操作の中心に配置し、「触れたくなる」「続けたくなる」体験を意識しました。

7. 成果・学び

本プロジェクトを通して、防災のような重いテーマでも、UX次第で行動のハードルを下げられることを学びました。
課題を正しく定義し直すことで、機能追加ではなく体験そのものを変えるアプローチが可能になります。
「役に立つ」だけでなく「続けたくなる」設計の重要性を、実感をもって理解できたプロジェクトです。
